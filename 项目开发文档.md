# 贵阳市房地产项目经济技术指标计算器 — 开发文档

## 项目概述

面向贵阳市房地产开发前期的经济技术指标自动生成工具。输入规划条件（容积率、密度、限高等）和产品定位参数，自动计算完整的《项目经济技术指标表》，包括计容面积、分业态面积、户数人口、车位、配套面积、地下面积、楼栋配置及可行性分析。

**技术栈：** 单文件 HTML5 + CSS + 原生 JavaScript，零依赖，浏览器直接运行。

**适用规范：**
- 《贵阳市城市规划管理技术规定（试行）》筑府办函[2024]9号
- 《城市居住区规划设计标准》GB50180-2018
- 《贵阳市新建改建居住区公共文化配套设施建设管理规定（暂行）》筑府发[2018]17号
- 《关于建设项目规划设置快递综合服务站指导意见（试行）》

## 核心计算引擎

### 计算链路

```
Step 1: 用地计算
  ├─ 计容用地面积 = 建设用地面积 + 平衡用地面积
  ├─ 建筑基底面积 = 计容用地 × 建筑密度
  ├─ 绿地面积 = 计容用地 × 绿地率
  └─ 道路广场 = 计容用地 - 基底 - 绿地

Step 2: 计容面积分解
  ├─ 计容总面积 = 计容用地 × 容积率
  ├─ 商业面积 = 计容总面积 × 商业占比
  ├─ 配套面积 = getFacilities(人口, 计容总面积, 商业面积, 户数) + 自定义配套 + 学校
  └─ 住宅面积 = 计容总面积 - 商业面积 - 配套面积

Step 3: 户数与人口
  ├─ 按产品配置（套数优先）推算总户数
  └─ 总人口 = 总户数 × 户均人口（默认 2.7）

Step 4: 车位计算
  ├─ 住宅车位 = 住宅面积/100 × 配建标准
  ├─ 商业车位 = 商业面积/100 × 配建标准
  ├─ 配套车位 = 配套面积/100 × 配建标准
  ├─ 地面车位 = 总车位 × 地面比例（≤10%）
  └─ 充电车位 / 人防车位 / 非机动车位

Step 5: 地下面积
  ├─ 地下车库 = 地下车位 × 35㎡/车位
  ├─ 人防面积 = 计容面积 × 人防比例
  ├─ 设备用房 = 地下车库 × 经验系数
  └─ 地下总面积汇总

Step 6: 配套设施（千人指标）
  ├─ 物业管理用房：分段线性公式
  ├─ 便利店：ceil(人口/3000) × 50㎡
  ├─ 快递服务站：ceil(商业面积/10000) × 30㎡ + ceil(户数/600) × 30㎡
  ├─ 文化设施：户数/人口分档匹配（图书室、电子阅览室、多功能室、文体活动室）
  └─ 其余按人口分段向下包含
```

### 产品配置计算（套数优先）

```
T4（4户/层，A×2+B×2）：
  加权户均 = (A户均 × A占比 + B户均 × B占比) / 100
  总套数 = round(产品面积 / 加权户均)
  A套数 = round(总套数 × A占比/100)
  B套数 = 总套数 - A套数
  A面积 = A套数 × A户均, B面积 = B套数 × B户均
  diff = 产品面积 - (A面积 + B面积)
  A面积 += diff  // 补齐差值

T3（3户/层，A×2+B×1）：
  默认 A:B = 67:33（即 2:1）
  其余逻辑同 T4
```

## 楼栋配置与可行性分析

### 均层法算法

```
总层数 = 产品面积 / 标准层面积
最少栋数 = ceil(总层数 / 限高层数)

遍历栋数 = 最少栋数 ... 最少栋数+3：
  均层数 = 总层数 / 栋数
  高F = ceil(均层数), 低F = floor(均层数)
  高栋数 = round((总层数 - 栋数 × 低F) / (高F - 低F))
  低栋数 = 栋数 - 高栋数
  选层差最小的方案
```

### 关键参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| 层高 | 3.0m | 标准层层高 |
| 首层高度 | 4.5m | 大堂/架空 |
| 女儿墙+高差 | 2.1m | 1.8+0.3 |
| 占地系数 | 1.10 | 可手动调整（标准层面积→占地面积） |
| 得房率 | 78%-90% | 按产品类型（高层78%/洋房82%/别墅90%） |
| 户均人口 | 2.7 人/户 | 可切换（2.6/2.7/2.8/自定义） |

### 输出内容

- 楼栋配置表（类型/楼型/户型/占地面积/标准层面积/层户数/层数/高度/单栋计容/栋数/总计容/总户数）
- 密度校验（测算密度 vs 设定密度）
- 商业层数对密度的影响对比
- 面积/户数核对
- 可行性结论与建议

## 项目管理

- **存储方式：** localStorage（浏览器本地存储）
- **数据结构：** JSON 数组，每个项目含 inputs + result 完整快照
- **支持操作：** 保存 / 加载 / 删除 / 导出 JSON 备份 / 导入 JSON 恢复
- **同名处理：** 自动覆盖更新

---

## 实时重算

所有输入框（规划条件、产品配置、学校、配套、车位、项目名称）变更后，300ms 防抖自动重算整表。基于 `input`/`change` 事件触发 `debouncedCalculate()`。

## 数据模型

### 输入关键字段

| 字段 | 类型 | 说明 |
|---|---|---|
| land_area | float | 总用地面积 (㎡) |
| construction_land_area | float | 建设用地面积 (㎡) |
| balance_land_area | float | 平衡指标用地面积 (㎡) |
| far | float | 容积率 |
| building_density | float | 建筑密度 (%) |
| green_ratio | float | 绿地率 (%) |
| height_limit | float | 建筑限高 (m) |
| planned_commercial_ratio | float | 商业计容占比 (%) |
| res_t4 / res_t3 / ... | float | 各产品占比 (%) |
| unit_t4a / unit_t4b / ... | float | 各户型户均面积 (㎡) |
| split_t4a / split_t4b / ... | float | T4/T3 子户型拆分比 (%) |

### 输出关键字段（Result 对象）

```javascript
{
  land: { land_area, construction_land_area, build_land, balance_land, footprint, green_area, road_hard },
  totalFar, resArea, comArea, totalBuilding,
  population: { units, population, personsPerHH },

  underground: { total, civilArea, nonCivilArea, civilPark, nonCivilPark },
  nonmotor: { total, ground, underground, area },
  customFacs: [...], schools: [...], inp: {...}
```

## 更新维护

### 配套设施规则更新

1. 编辑 `配套设施标准.xml`（Excel 另存为 XML 电子表格）
2. 运行 `python update_rules.py`
3. 自动生成 `facilities.js`

### 楼栋分析更新

直接编辑 `building_analysis.js`，独立模块不影响主文件。
